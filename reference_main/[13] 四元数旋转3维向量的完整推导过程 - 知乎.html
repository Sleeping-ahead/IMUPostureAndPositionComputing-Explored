FORCEINLINE FVector FQuat::RotateVector(FVector V) const
{	
	// http://people.csail.mit.edu/bkph/articles/Quaternions.pdf
	// V' = V + 2w(Q x V) + (2Q x (Q x V))
	// refactor:
	// V' = V + w(2(Q x V)) + (Q x (2(Q x V)))
	// T = 2(Q x V);
	// V' = V + w*(T) + (Q x T)

	const FVector Q(X, Y, Z);
	const FVector T = 2.f * FVector::CrossProduct(Q, V);
	const FVector Result = V + (W * T) + FVector::CrossProduct(Q, T);
	return Result;
}
引言：一直没弄明白四元数是如何旋转Vector的，对于四元数也只是会用，偶然在虚幻代码中看到了这么一段代码，就抽时间完整的研究了一遍所有的推导过程，本文章全程使用右手坐标系，但是虚幻是左手坐标系，并不影响推导，过程在附件。

1.最近又复习了一下，有些重要的点记一下：

1、三维向量其实相当于实数部为0的四元数，即纯四元数。在R上满足代数除法的数，只有1，2，4，8四个维度。四元数群已经不满足乘法交换律了，而到了八元数连乘法结合律也不满足了。

2、复数的旋转是乘以一个单位向量cosθ+isinθ,i其实就是实数部为零的复数，扩展到四元数的旋转这个i就是一个纯四元数。四元数的乘法打出来太麻烦，可以参见我的手写推导。另外有一个非常神奇的事情就是纯四元数相乘，实部是向量的点积，虚部是向量的叉积。

3、因为想要四元数表示旋转，所以三维向量也就是纯四元数做完运算之后，长度维度都不变，不能做完旋转，三维向量跑四维平面去了，所以需要左乘右乘四元数，这两个四元数应该互逆（共轭）,手写推导过程参见附件。而且应当是单位四元数，否则会产生缩放。

4、需要注意，根据利用欧拉公式的推导分解为左乘右乘之后要用半角，具体推导可看附件，也就是说想要旋转θ，被分解成左乘和右乘之后，是V' =qvq-1= cos(θ/2)+sin(θ/2)(r)*v*cos(-θ/2)+sin(-θ/2)(r)。r代表转轴，应当是单位化的三维向量。四元数可视化网址如下，比较形象的理解就是先把三维向量超到4维再拉回来。

2.实际应用

比如在UE4中如果想要让一个物体绕着Z轴，旋转-45度，写法应当是FQuat(FVector(0.f, 0.f, -1.f), PI * 0.25f)，等价于欧拉角（0,0,-45).

简单的理解就是先算出一个单位旋转轴，然后旋转多少度，正负号是顺时针还是逆时针。

FORCEINLINE FQuat::FQuat(FVector Axis, float AngleRad)
{
	const float half_a = 0.5f * AngleRad;
	float s, c;
	FMath::SinCos(&s, &c, half_a);

	X = s * Axis.X;
	Y = s * Axis.Y;
	Z = s * Axis.Z;
	W = c;

	DiagnosticCheckNaN();
}
